About Routing

  1. All the routes/paths or navigation requirements in amgular are handheld by angular router package
  2. We navigate from one component view to other using the routes
  3. we can configure various types of routes
    • default routes
    • child routes
    • wild card routes
    • query params
    • url segments
    • lazy loading
  4. We create a routes array and whenever user requests a toute it will search in the routes array
  5. Router is Singleton - which means there is only one instance of angular router per application

Angular Routing - Routing Strategies
  1. This is extremely important as it ill affect the entire application flow and navigation
    • /home
    • /dashboard
    • /search?keyword=k1
    • /product/10
    • /product/10/images
  3. PathLocation or HashLocation
  4. We need to add this in Providers of our module
    • {{'{'}}provide:LocationStrategy, useClass:HashLocationStrategy{{'}'}}
  5. By default - Angular makes use of the PathLocationStrategy
  6. With HashLocationStrategy - we will see the # in the URL
  7. By default all Angular Apps are using PathLocationStrategy
    1. /home
    2. /products
    3. /search

    • HashLocation
    1. /#/home //HashLocation
    2. /#/products //HashLocation
  8. We can choose any strategy we want for our app
  9. There is no constraint or restriction

Routing - Base HREF
  • Every Angular application has MANDATORY base HREF
  • Angular application is a SPA(Single Page Architecture) which means there will be only one HTML file
  • The default base href is set ti "/" the root folder
  • The Base HREF is present in index.html file for all Angular applications
