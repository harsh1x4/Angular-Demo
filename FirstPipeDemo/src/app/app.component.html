Pipes

  1. Pipes are used to transform data.
  2. Pipes will take data input and convert/transform into a desired form.
  3. Pipes are written using the pipe operator(|)
  4. We can apply pipes to any view/template and to any data input

Types of Pipes

  • Built in Pipes
    • Lowercase
    • Uppercase
    • Currency
    • Date
  • Parametrized Pipes
    • We can pass one or more parameters to pipes
  • Chaining Pipes
    • We can connect multiple pipes to a data input
  • Custom Pipes
    • We can create our own custom pipes for various data formatting.

Built In Pipes & Parameterizing Pipes

  • Lowercase
  • Uppercase
  • Percent
  • Currency
  • Date
  • JSON
Lowercase Pipe :- {{user.firstName | lowercase}},
Uppercase Pipe :- {{user.lastName | uppercase}}
Date Pipe :- {{user.today | date}}
Currency Pipe Without Code :- {{user.salary | currency:'INR'}}
Currency Pipe With Code :- {{user.salary | currency:'INR':'code'}}
JSON pipe :- {{user | json}}

Chaining Pipes

  • Using multiple pipes on a data input is called as Chaining Pipes
  • We can pass one or more pipes to a data
Using two Pipes simultaneously :- {{user.today|date|uppercase}}

Custom Pipes

  • We can create our own custom pipes for various data formatting.
  • We can generate a new custom pipe using the CLI
    Ng generate pipe pipe_name
  • Pipes needs to be added to the module - CLI does this when we auto generate
  • We need to import the Pipe and PipeTransform from the @angular/core
  • Pipes are declared with the decorator @pipe and provide the selector name
Square Root of 225:{{225 |sqrtTest}}
Square Root of 25:{{25 |sqrtTest}}
Square Root of 169:{{169 |sqrtTest}}
